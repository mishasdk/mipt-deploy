# Постановка задачи: предсказание оттока клиентов

## Контекст

Датасет: Telco Customer Churn (7 043 клиента, 21 признак, ~26.5% оттока).
Заказчик - телеком-оператор. Клиенты уходят к конкурентам, узнаём об этом постфактум.

## Бизнес-задача

Заранее находить клиентов, готовых уйти, чтобы отдел удержания делал им предложение до ухода.

## Бизнес-метрика

Churn rate - доля ушедших клиентов за месяц.

## ML-задача

| | |
|---|---|
| Тип | Бинарная классификация |
| Target | `Churn` (1 - ушёл, 0 - остался) |
| Признаки | Данные о пользователе |
| Инференс | Батч (раз в сутки) |
| Выход | Вероятность оттока + top-K клиентов для кампании |

## ML-метрики

Primary: PR-AUC - класс несбалансирован, ROC-AUC завышает оценку.
Secondary: Recall@K - отдел удержания обрабатывает фиксированное число клиентов в месяц, нужно поймать максимум уходящих в этот лимит.
