# 2. latency-improvements

Date: 2026-05-31

## Status

Accepted

## Context

Мы сравниваем два набора данных по вермени отклика (latency) 
равниваются два набора измерений latency инференс сервиса:
- У существующей системы avg latency 3.5, среднее отклонение 0.4.
- У улучшенной 2.0, std 0.4.
- размер обоих выборок 500_000.

Требуется принять решение о выкатке улучшенной модели, на основе MDD.

## Decision

Решение о выкладке должно приниматься на основе статистического теста, если среднее время новой системы больше старой на уровне значимости 0.05 то выкатываем.

Гипотезы:
- H_0: среднее время отклика улучшенной системы >= существующей (улучшения нет).
- H_1: среднее время отклика улучшенной системы < существующей (улучшение есть).

Выборки независимы, дисперсия и матожидаение известны - используем Welch t-test. Уровень значимости a = 0.05.

Результат:
- t = −1875, p-value < 1e−300 (p < α).
- d = 3.75 (очень большой эффект).
- Разница средних: 1.5 с (−43 %).

H_0 отклоняется. У улучшенной системы latency ниже.

Улучшенную систему выкатываем в прод, по стратегии blue/green с переключением трафика (5, 10, 20, 50, 80, 100), старую модель выводим из эксплуатации.

## Consequences

- latency снижается с 3.5 до 2 - выигрышь по метрике latency.
- в случае проблем во время раскатки новой версии модели эксперимент останавливается и делается rollout на старую версию.
