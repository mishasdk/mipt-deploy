# Шаг 4. Управление рисками: SLI и SLO

Для каждого компонента задаём **SLI** (что измеряем) и **SLO** (целевой/критический уровень нормальной работы).

SLI разнесены по трём уровням: **технический**, **модельный**, **бизнес**.

---

## 1. Технический уровень

Инфраструктура и сервисы: ML-сервис (FastAPI), Nginx-прокси, Airflow, Feature Store
(Postgres), MLflow/S3, VM, CI/CD.

| Компонент | SLI | SLO (норма / критич.) |
|---|---|---|
| ML-сервис `/predict` | Доступность = доля ответов без 5xx | >= 99.5% / <99% > |
| ML-сервис `/predict` | Latency, p95 | < 10 c / > 20 c |
| ML-сервис `/predict` | Доля ошибок (5xx + таймауты) | < 1% / > 3% |
| Nginx-прокси | Доступность шлюза (без 5xx) | >= 99.9% / < 99.5% |
| Nginx-прокси | Накладной latency проксирования, p95 | < 20 мс / > 50 мс |
| Airflow DAG обучения | Доля успешных запусков DAG | >= 99% / < 95% |
| Airflow DAG обучения | Задержка старта от расписания | < 5 мин / > 30 мин |
| Feature Store | Доступность чтения фич | >= 99.5% / < 99% |
| Feature Store | Latency чтения, p95 | < 50 мс / > 150 мс |
| Feature Store | Свежесть фич (лаг суточной выгрузки) | < 1 ч / > 6 ч |
| VM | CPU (устойчивая загрузка) | < 80% / > 90% |
| VM | RAM | < 85% / > 95% |
| VM | Диск | < 85% / > 90% |
| Monitoring (Prometheus) | Доля успешных scrape таргетов | >= 99% / < 95% |
| CI/CD (GitHub Actions) | Доля успешных деплоев | >= 95% / < 85% |
| CI/CD (GitHub Actions) | Время отката (rollback / MTTR) | < 10 мин / > 30 мин |

## 2. Модельный уровень

Качество модели на проде.

| SLI | Описание | SLO (норма / критич.) |
|---|---|---|
| PR-AUC (realized) | Качество на дозревших метках, primary | >= baseline - 5% / < baseline − 10% |
| Recall@500 (realized) | Доля пойманных уходящих в лимите кампании | >= 0.60 / < 0.50 |
| PSI предсказаний | Дрейф распределения скоров vs обучение | < 0.2 / > 0.25 |
| Data drift (входные фичи) | Доля фич с PSI > 0.2 | < 10% / > 25% |
| Доля валидных предсказаний | Без NaN / вне диапазона [0,1] | >= 99.9% / < 99% |
| Свежесть модели | Время с последнего успешного переобучения | < 7 дней / > 14 дней |

## 3. Бизнес-уровень

Влияние системы на бизнес-цель — удержание клиентов.

| SLI | Описание | SLO (норма / критич.) |
|---|---|---|
| Конверсия удерживающего предложения | Доля принявших оффер из обработанных top-K | >= 15% / < 8% |
| Churn rate среди обработанных | Доля ушедших из тех, кому сделали предложение | <= baseline − 3 п.п. / >= baseline |
| Покрытие кампании | Доля целевых клиентов, получивших оффер вовремя | >= 95% / < 80% |
| Возвраты/retention | Доля удержанных клиентов за месяц | >= целевого / < целевого − 5 п.п. |
