# ML-манифест: Customer Churn Prediction

## 1. Предыстория

Телеком-оператор теряет клиентов. Узнаём об уходе постфактум — повлиять уже нельзя. Привлечение нового клиента дороже удержания старого.

## 2. Ценностное предложение

Модель находит клиентов с высоким риском оттока до их ухода. Отдел удержания делает таргетированное предложение, тем самым сохраняем выручку.

## 3. Цели

- Найти клиентов с высоким риском оттока на горизонте 1 месяца
- Ранжировать по приоритету (top-K для кампании)
- Автоматически переобучаться при деградации качества

## 4. Решение

**Заявленный уровень зрелости: 2**

Система включает компоненты:

| # | Компонент | Реализация |
|---|---|---|
| 1 | Автоматический пайплайн обучения | Airflow DAG |
| 2 | Версионирование данных и моделей | DVC, MLflow Model Registry |
| 3 | Хранилище датасетов и моделей | S3 object storage |
| 4 | Feature Store | Postgres — расчёт и хранение фич, чтение при инференсе |
| 5 | Прокси с API gateway | Nginx |
| 6 | Мониторинг качества модели на проде | Prometheus, Graphana |
| 7 | CI/CD для деплоя | GitHub Actions |

## 5. Осуществимость

Данные из открытого датасет Telco Customer Churn (7 043 записи, 21 признак, ~26.5% оттока)

## 6. Данные

- Источник обучающего набора: Telco Customer Churn (CSV под DVC)
- Прод-данные: синтетические свежие данные генерируются скриптом раз в сутки (имитация выгрузки данных из прод БД)
- Разметка: target `Churn` уже размечен в датасете; на проде разметка приходит с задержкой (например, 30 дней - факт ухода)

## 7. Метрики

- Бизнес: churn rate
- ML метрика: Recall@K

## 8. Оценка качества модели

Офлайн:
- PR-AUC (primary)
- Recall@500
- Brier score

Онлайн:
- PSI распределения предсказаний (детектор drift)
- Realized PR-AUC (с задержкой 30 дней)
- p95 latency `/predict`

## 9. Подбор модели

Итеративно:
1. Baseline: Logistic Regression
3. Random Forest
2. Gradient Boosting (LightGBM) — основной кандидат

## 10. Инференс

Батчевый - раз в сутки получаем список пользователей, которые потенциально могут уйти.

## 11. Обратная связь

Источники:
- Факт ухода/удержания клиента
- Конверсия удерживающего предложения
- Метрики дрейфа данных и предсказаний

MDD — используется, переключение версий модели и архитектурные решения принимаются на основе статистических тестов на метриках.
